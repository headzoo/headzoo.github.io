React Router Link Delay

July 30 2017 Â· 2 min read

I noticed when I started writing single page apps, that with small sites, or static sites which don't make backend requests, clicking on React Router's Link component renders the next page so quickly that the experience is somewhat jarring.

We're accustomed to waiting a moment after clicking a link before the next page comes up, which gives our eyes a moment to change focus back to the center of the screen in anticipation of the next page. Without such a delay the transition from one page to the next is slightly unpleasant.

Adding a fading transition is one way to make the DOM change less jarring. Another is adding a small artificial delay after clicking the link before rendering the next page. For that I have a component which decorates the React Router Link component, and adds a small delay before pushing the next route into the browser history. I set the delay to 150 milliseconds on my static sites.

import React from 'react';
import PropTypes from 'prop-types';
import { Link } from 'react-router-dom';
/**
* Wraps the React Router Link component and creates a delay after the link is clicked.
*/
export default class DelayLink extends React.Component {
static propTypes = {
/**
* Milliseconds to wait before registering the click.
*/
delay: PropTypes.number,
/**
* Called after the link is clicked and before the delay timer starts.
*/
onDelayStart: PropTypes.func,
/**
* Called after the delay timer ends.
*/
onDelayEnd: PropTypes.func
};
static defaultProps = {
delay: 0,
onDelayStart: () => {},
onDelayEnd: () => {}
};
static contextTypes = Link.contextTypes;
constructor(props) {
super(props);
this.timeout = null;
}
componentWillUnmount() {
if (this.timeout) {
clearTimeout(this.timeout);
}
}
/**
* Called when the link is clicked
*
* @param {Event} e
*/
handleClick = (e) => {
const { replace, to, delay, onDelayStart, onDelayEnd } = this.props;
const { history } = this.context.router;
onDelayStart(e, to);
if (e.defaultPrevented) {
return;
}
e.preventDefault();
this.timeout = setTimeout(() => {
if (replace) {
history.replace(to);
} else {
history.push(to);
}
onDelayEnd(e, to);
}, delay);
};
render() {
const props = Object.assign({}, this.props);
delete props.delay;
delete props.onDelayStart;
delete props.onDelayEnd;
return (
<Link {...props} onClick={this.handleClick} />
);
}
}

Use it as follows: 

<DelayLink to="/some/route" delay={150} />

The link waits 150 milliseconds before rendering the next page. The component also includes two callbacks: onDelayStart(), and onDelayEnd(). I use them to animate the icon next to the links in this site's nav bar.
